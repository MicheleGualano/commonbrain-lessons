Gate on gosec's exit code, not its ANSI-colored summary line
#go #gosec #ci #ansi #gotcha
Rule. gosec's `Issues : N` summary is ANSI-colored (the number is wrapped in escape codes ending in the reset `[0m`). Parsing it with something like `grep -oE '[0-9]+' | tail -1` grabs the 0 from that reset code, NOT the issue count, so the gate reports zero for everything and masks real findings. For any pass/fail gate, use the tool's EXIT CODE (0 means clean). If you must read a number, strip ANSI first and anchor to the right line. General rule: distrust a checker that always reports the same good value — prove it works by feeding it a case that MUST fail.

Triggers

  • gosec reports 0 issues but there are findings
  • parsing gosec Issues line returns 0 or the wrong number
  • grep -oE '[0-9]+' | tail -1 picks up the ANSI reset code
  • lint sweep said clean but the gate failed
  • reading a colored CLI summary line for a count
