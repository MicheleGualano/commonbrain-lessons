Knowledge for humans AND agents: split into an explain layer and a retrieve layer
#knowledge #docs #agents #retrieval #design
Rule. Humans browse and read; agents grep raw text, where prose markup is noise. Build two layers: a rich EXPLAIN layer (prose, examples, diagrams) for understanding, and a compact RETRIEVE layer of structured records for finding. Key each retrieve record on LITERAL triggers (the exact error strings and symptoms you would face), a one-line rule, and a pointer back to the explain file. Optimize for found-in-one-second, not for looking nice.

Triggers

  • building docs or a knowledge base consumed by both a human and an LLM agent
  • a knowledge base exists but the agent never queries it
  • how should I structure notes so a future agent finds them fast
  • agent greps raw files and HTML or markup is too noisy to search
