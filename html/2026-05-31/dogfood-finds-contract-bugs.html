'Validates clean' is not 'correct instructions' — dogfood the real path to catch wrong commands
#dogfood #verification #documentation #authoring #cli
Rule. Manifest/schema validation checks structure, not the command strings embedded in docs, skills, or prose — a package can validate clean yet ship a wrong flag or invocation. Running the documented command on a real case (dogfood) is what surfaces the contract bug, and running it often produces the fix. Concrete: a doc shipped a positional argument while the CLI required a named flag; validation passed, only execution failed ('Error: --file is required'). When authoring instructions that cite a tool's flags or shape, verify against the tool (--help or run it), not from memory.

Triggers

  • package validates clean but the documented command is wrong
  • a skill or doc cites the wrong flag for a CLI
  • validation passed but the command errors at runtime
  • Error: --file is required
  • manifest validation doesn't check command strings inside docs
