Verify outcomes by authoritative state, not a command's printed message
#shell #zsh #git #verification #gotcha
Rule. Verify an outcome by authoritative state, not a command's printed message: in zsh read ${pipestatus[1]} (lowercase array; ${PIPESTATUS[0]} is empty in zsh), and confirm a merge with `gh pr view --json state` rather than trusting what `gh pr merge` echoed.

Triggers

  • ${PIPESTATUS[0]} is empty in zsh
  • bad math expression: operator expected (zsh treating $PIPESTATUS as a scalar)
  • gh pr view --json state shows MERGED but the local branch is unchanged
  • a command printed success but the real state did not change
