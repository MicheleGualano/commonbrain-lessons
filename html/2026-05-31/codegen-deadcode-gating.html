Gate a generated definition on the OR of its real call-site conditions, plus its imports
#codegen #dead-code #linter #imports #gating
Rule. When a generator emits a definition under condition A but its call sites are gated under B and C (both strict subsets of A), output shapes in A but not in B or C ship the definition with no caller, so the linter flags it dead. Gate the definition on (B || C), the exact OR of the real call-site conditions, not a convenient superset. Then gate the orphaned imports on that same condition: a format-only import pass (goimports/gofmt without removal) reorders but does not strip them, giving an 'imported and not used' compile error. Compile and lint every shape, not just the convenient one.

Triggers

  • func X is unused (unused)
  • X declared and not used
  • imported and not used after removing or gating a function in a template
  • generated code has dead functions only in some feature/auth shape
  • goimports FormatOnly did not strip the orphaned import
