A Claude Code statusLine command fails silently on empty stdin or a missing jq
#claude-code #statusline #jq #silent-failure #stdin
Rule. A statusLine of type command reads a JSON payload on stdin and prints one line. Two silent-failure traps make a correct script look broken: (1) testing with echo '{}' gives empty output because every field (.workspace.current_dir, .model.display_name, ...) is absent so each guarded append skips — wrong test, not a bug; (2) if jq is missing from PATH every extraction returns empty, so the line is blank every launch. Before calling a stdin-fed hook or status script broken, verify command -v jq and feed it a realistic payload with the real fields; empty input is never a valid test case.

Triggers

  • status line appears blank or inconsistent across sessions
  • echo '{}' | bash statusline.sh prints nothing
  • statusLine command produces no output
  • status line doesn't show up every time
