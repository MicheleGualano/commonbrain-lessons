staticcheck ST1005: error strings stay lowercase with no trailing punctuation
#go #staticcheck #st1005 #errors #style
Rule. Error strings should start lowercase and end without a period, colon, or other punctuation, e.g. `fmt.Errorf("profile %q not found")`. The reason is that errors get wrapped and concatenated into larger messages, so they must read as a clause, not a standalone sentence. Proper nouns and acronyms (HTTP, JSON) keep their case; only the leading ordinary word is lowercased.

Triggers

  • ST1005: error strings should not be capitalized
  • ST1005: error strings should not end with punctuation
  • staticcheck flags a capitalized fmt.Errorf message
