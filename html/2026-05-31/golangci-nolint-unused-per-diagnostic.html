//nolint:unused is a per-line filter, not reachability — annotate the whole dead tree
#go #golangci #unused #nolint #static-analysis
Rule. staticcheck computes `unused` by call-graph reachability; golangci-lint applies `//nolint:unused` afterwards as a per-diagnostic line filter. So a whole dead subtree (a root plus every helper only it calls) is flagged, and nolint on the root does NOT mark the leaves used — annotate EVERY symbol in the tree. Legitimate only for intentionally-kept code (e.g. an affordance a test asserts is emitted but never calls). If the symbol is used in some build shapes, gate its definition instead, else nolintlint reports the directive itself as unused where the symbol is live.

Triggers

  • func X is unused (unused) but it is an intentionally-kept helper
  • //nolint:unused did not clear the transitive dead functions it calls
  • nolintlint directive `//nolint` is unused for linter
  • should I nolint or gate a sometimes-used helper
