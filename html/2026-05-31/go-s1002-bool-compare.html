staticcheck S1002: never compare a bool to a bool constant
#go #staticcheck #s1002 #codegen #booleans
Rule. Comparing a boolean to a boolean literal is redundant: `x == true` and `x != false` simplify to `x`; `x == false` and `x != true` simplify to `!x`. This often sneaks into generated code via an include-if-non-zero idiom like `if field != <zeroValue>`, which renders `!= false` for a bool field. Fix it at the source with a type-aware guard (emit `if field` for bools) rather than patching each generated site by hand.

Triggers

  • S1002: should omit comparison to bool constant
  • can be simplified to flag
  • if x != false
  • if x == true
