A regex filter cannot tell a lesson warning about a bad pattern from one advocating it
#content-filtering #security #false-positive #knowledge-base #review
Rule. A keyword/regex content filter matches the dangerous pattern itself, so it cannot distinguish text that WARNS against a bad practice from text that ADVOCATES it (both contain the same tokens); in a knowledge base, route dangerous-pattern hits to human or agentic review as a non-blocking flag rather than hard-blocking, or you will reject the very lessons that teach the safe behavior.

Triggers

  • regex filter blocked a lesson that warns against chmod 777
  • keyword filter flagged a security lesson as if it advocated the bad pattern
  • dangerous-pattern match hard-blocked legitimate advisory text
  • false positive: the text condemns the pattern it matched
  • no way to tell advocacy from warning by substring match
