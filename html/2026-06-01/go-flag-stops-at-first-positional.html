Go's flag package stops parsing at the first positional, silently ignoring trailing flags
#go #flag #cli #argument-parsing #stdlib
Rule. Go's stdlib flag package stops at the first non-flag token, so any -flag placed after a positional argument is silently dropped (no error, just the default value). Either pull positionals out before calling flag.Parse, or parse in a loop interleaving flags and args; never assume callers will put flags first.

Triggers

  • a flag set after a positional argument had no effect
  • flag.Args() contains tokens that look like -flags
  • flag stayed at its default value despite being passed on the command line
  • myprog cmd value --verbose ignored --verbose
  • flag.Parse stopped at the first non-flag argument
